# Model Card

For additional information see the Model Card paper: https://arxiv.org/pdf/1810.03993.pdf

## Model Details

- **Model type:** Random Forest Classifier (`sklearn.ensemble.RandomForestClassifier`,
  `n_estimators=100`, `max_depth=10`, `random_state=42`).
- **Developed by:** Project author, as part of the Udacity "Deploying a
  Scalable ML Pipeline in Production" (nd0821-c3) course project.
- **Inputs:** 13 demographic/work attributes from the UCI/Census Bureau
  "Adult" dataset (age, workclass, fnlgt, education, education-num,
  marital-status, occupation, relationship, race, sex, capital-gain,
  capital-loss, hours-per-week, native-country). Categorical features are
  one-hot encoded; continuous features are left unscaled.
- **Output:** Binary classification of annual salary as `<=50K` or `>50K`.
- **Version:** 1.0.0

## Intended Use

The model is intended for educational purposes, to demonstrate an
end-to-end ML pipeline: training, unit testing, slice-based validation,
deployment behind a FastAPI service, and a GitHub Actions CI/CD pipeline.
It is **not** intended for use in any real-world decision-making process
(e.g., lending, hiring, benefits eligibility), since the underlying data is
from a 1994 Census Bureau extract and the model has not been audited for
fairness or robustness for production use.

## Training Data

The model was trained on an 80% split (26,048 rows) of the cleaned
`census.csv` dataset (32,561 rows total, sourced from
https://archive.ics.uci.edu/ml/datasets/census+income). Whitespace
surrounding column names and string values in the raw file was stripped
(`data/clean_data.py`) before processing. Categorical features were
one-hot encoded and the label was binarized using scikit-learn's
`OneHotEncoder` and `LabelBinarizer`, fit only on the training split.

## Evaluation Data

The remaining 20% (6,513 rows) was held out as a test set and processed
using the encoder/label binarizer fit on the training data only, to avoid
data leakage.

## Metrics

Overall performance on the held-out test set:

| Metric    | Value  |
|-----------|--------|
| Precision | 0.7962 |
| Recall    | 0.5372 |
| F1        | 0.6416 |

Performance was also computed on slices of each categorical feature (see
`slice_output.txt`, generated by `train_model.py`). A few notable slices:

| Feature: sex | n    | Precision | Recall | F1     |
|--------------|------|-----------|--------|--------|
| Female       | 2126 | 0.7949    | 0.3991 | 0.5314 |
| Male         | 4387 | 0.7964    | 0.5613 | 0.6585 |

Precision is comparable across sexes, but recall for the `Female` slice is
notably lower than for `Male` (0.40 vs 0.56) — see Ethical Considerations.

## Ethical Considerations

- The recall gap by `sex` shown above means the model is more likely to
  miss a true `>50K` outcome for women than for men. This kind of
  disparity is a known issue with the Adult/Census Income dataset, which
  reflects historical labor-market patterns (including a 1994 gender pay
  gap) rather than a fair underlying distribution.
- Other slices (e.g. `workclass`, `education`, `native-country`) also show
  uneven sample sizes and metric variance — see `slice_output.txt` for
  the full breakdown before drawing conclusions from any single slice.
- Because predictions are tied to sensitive demographic attributes (sex,
  race, native-country), this model should not be used to make or inform
  decisions about real individuals.

## Caveats and Recommendations

- The dataset is from a 1994 Census Bureau extract and does not reflect
  current economic conditions, cost of living, or labor markets.
- Class imbalance (more `<=50K` examples than `>50K`) likely contributes
  to the moderate recall observed; resampling or class-weighting could be
  explored to improve it.
- For any use beyond this educational exercise, the model would need a
  fairness audit across slices (especially `sex`, `race`, and
  `native-country`), more recent and representative training data, and a
  clearly scoped, non-decision-making use case.
